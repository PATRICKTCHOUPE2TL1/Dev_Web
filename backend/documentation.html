DOCUMENTATION API

Introduction

Cette documentation devrait vous aider à vous familiariser avec les ressources disponibles et comment les utiliser grace a des requêtes HTTP.Cette API nous permet d'effectuer facilement des requettes depuis notre base de données

Commencer

Pour effectuer des requête vous pouvez soit :

  • utiliser la commande curl dans un terminale exemple : curl http://127.0.0.1:5000/utilisateur
  • utiliser postman : il vous suffit juste d'entrer l'url dans la barre d'address qui ce trouve sur la page d'accueil.
  • Vous pouvez juste entrer l'url dans un navigateur pour consulter la resource

Resources

vous trouverez ici toutes les operations que nous pouvons effectuer sur cette api

  • consulter toutes les données disponible dans la base de données: ceci peut etre fait en accedant a l'url racine http://127.0.0.1:5000
  • consulter les données d'une table specifique :le format pour cette requette est la suivante : url_racine/nom_table exemple http://127.0.0.1:5000/utilisateur
  • consulter juste une colone d'une table specifique :le format por cette requette est la suivante: url_racine/nom_table/nom_colone exemple http://127.0.0.1:5000/utilisateur/prenom
Il n'ya pas de limite en cequi concerne le nombre de requette que vous pouvez effectuer sur cette api

Recherche

vous pouvez effectuer des reserches plus approfondis grace en ajoutant /search?
par exemple :
si nous souhaitons avoir le id d'un medecin qui reside a paris, nous pouvons le faire grace a la requette suivantes
http://127.0.0.1:5000/medecin/nom/search?args1=localisation&args2=paris

si nous souhaitons plutot avoir toutes les informations sur ce medecin et pas seulement son id, on peut effectuer la requette suivantes
http://127.0.0.1:5000/medecin/search?args1=localite&args2=paris
